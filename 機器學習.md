# 機器學習（Machine Learning, ML）
- 機器學習（Machine Learning, ML）是人工智慧（AI）的一個子領域
- 核心目標是讓電腦系統能夠從數據中學習規律，並在不經由明確程式指令的情況下，對未知數據做出預測或決策。
- 簡單來說，傳統程式設計是「人工輸入規則 + 數據 = 結果」；而機器學習則是「數據 + 結果 = 自動產生的規則」。

# 機器學習的主要類型
- 根據學習方式與數據特性的不同，機器學習主要可分為以下四大類：
- `1`. 監督式學習 (Supervised Learning)
  - 這是目前應用最廣泛的類型。模型在學習時，每筆數據都帶有明確的**「標籤(答案)」（Label）**，即電腦知道正確答案。
  - 核心邏輯： 給予輸入 與對應的輸出 ，讓模型學習映射函數 。
  - 常見任務：
    - 分類 (Classification)： 預測類別（如：判斷這封郵件是否為垃圾郵件）。
    - 迴歸 (Regression)： 預測連續數值（如：根據坪數預測房價）。
- `2`. 非監督式學習 (Unsupervised Learning)
  - 數據沒有標籤
  - 模型必須自行找出數據中隱藏的結構或規律。
  - 核心邏輯： 探索數據的`內在分佈`或`相似性`。
  - 常見任務：
    - 分群 (Clustering)： 將相似的對象分在一起（如：根據購買行為進行客戶分群）。
    - 降維 (Dimension Reduction)： 簡化複雜數據，保留最重要的特徵（如：PCA 主成分分析）。
    - 關聯規則
- `3`. 半監督式學習 (Semi-supervised Learning)
  - 介於上述兩者之間。
  - 訓練數據中只有少部分有標籤，大部分則沒有標籤。
  - 這在標記成本高昂（如醫學影像判讀）的情況下非常實用
  - 模型會利用少量的標籤來引導對大量無標籤數據的理解。
- `4`. 強化學習 (Reinforcement Learning, RL)
  - 透過**「試錯」（Trial and Error）**來學習的方式。
  - 模型（Agent）在環境中執行動作，根據結果獲得「獎勵」或「懲罰」
  - 目標是最大化長期累積的獎勵。
  - 核心邏輯： 學習在特定狀態下採取最佳動作的策略（Policy）。
  - 常見應用： AlphaGo 圍棋、自動駕駛、機器人控制、遊戲 AI。
 
# 機器學習的生命週期 (ML Workflow)
- 要建立一個有效的模型，通常需要經過以下步驟：
```
1.	資料收集與清理： 處理缺失值、雜訊。
2.	特徵工程： 挑選對預測最有幫助的數據維度。
3.	模型訓練： 選擇合適的演算法進行運算。
4.	模型評估： 使用測試集確認準確度（如： 或均方誤差 ）。
5.	部署與優化： 將模型上線並根據新數據持續迭代。
```
